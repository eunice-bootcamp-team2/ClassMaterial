1️⃣ 기본 개념 — “폴더는 나무 구조”

```
루트(최상위)
 ├── pr
 │    ├── child
 │    └── docs
 └── etc
```

지금 위치(현재 폴더)를 👉 **pr/child** 라고 가정할게요.

- **루트(최상위)** = 가장 위
- 그 안에 `pr`, `etc` 같은 폴더가 있고
- `pr` 안에는 다시 `child`, `docs` 가 있어요.

---
###### 2️⃣ 한 단계 위로 가기 — `cd ..`
|상황|결과|
|---|---|
|현재 위치|`pr/child`|
|명령어|`cd ..`|
|이동 후|`pr/` 로 올라감|
공통 — 윈도우/리눅스 모두 동일
```bash
cd ..
```
➡ “부모 폴더로 이동” 이라는 뜻

---
3️⃣ 현재 폴더 보기 : 내가 어디 있는지 확인 — `pwd`
```bash
pwd
```

예:
```
/home/youjung/pr/child
```
➡ **현재 위치를 정확히 보여줍니다.**
> 💡 “길 잃었다 싶으면 `pwd` 부터!”

---
4️⃣ 같은 단계 폴더로 이동 (형제 폴더)
현재: `pr/child`  
가고 싶은 곳: `pr/docs`
```bash
cd ../docs
```
✔ 먼저 `..` 로 **위로 올라간 뒤**,  
✔ 그 안에서 `docs` 로 들어가는 흐름이에요.

> 📌 머릿속 이미지  
> child ⬅ (한 칸 위) pr ➜ docs

---
5️⃣ 최상위 루트(제일 위)로 가기
```bash
cd /
```
➡ 컴퓨터 전체의 **최상위 시작점**으로 갑니다.

---
6️⃣ 집(Home) 폴더로 가기 - ~
```bash
cd ~
```
`~` 는 **내 계정 폴더(내 방)** 을 뜻해요.

| OS                 | `~` 의 실제 위치      |
| ------------------ | ---------------- |
| Linux              | `/home/사용자이름`    |
| macOS              | `/Users/사용자이름`   |
| Windows PowerShell | `C:\Users\사용자이름` |
예:
```
~  ➜  /home/youjung
```
💬 그냥 이렇게 기억하세요  
**`~` = 내 방**

---
7️⃣ **다른 폴더로 바로 점프 (절대 경로)**
현재 위치가 어디든 상관없이 **그 위치로 바로 이동**
```bash
cd /home/user/projects
```
➡ “그 주소로 직행”

> 💡 집 주소처럼 **앞에서부터 끝까지** 적으면  
> 어디서든 정확히 이동합니다.

---
8️⃣ **상대 경로 vs 절대 경로**
상대 경로 (현재 위치 기준)
```bash
cd ..
cd ../docs
cd ../child/test
```
✔️ **절대 경로는 “집 주소”처럼 전체 경로를 다 적음**  
✔️ **상대 경로는 “지금 있는 곳에서” 이동**

➡ “지금 있는 자리”를 기준으로 계산

---
9️⃣ 파일이 “어디에 만들어졌는지(전체 경로)” 확인하는 방법

현재 위치 확인
```bash
pwd
```
예시결과:
```
/home/youjung/pr/child
```

그 경로 + 파일 이름을 합치면:

> `/home/youjung/pr/child/내파일.py`

👉 즉, **`pwd`가 가리키는 위치에 파일이 있는 것**이에요.

---
파일이 확실히 그 폴더에 있는지 확인
```bash
ls
```

또는
```bash
ls -l
```
목록에 나오면  
➡ **“지금 폴더에 만들어진 것”** ✔️

---
파일의 전체 경로를 한 줄로 확인
```bash
realpath 파일이름
```
예:
```bash
realpath mynote.py
```
출력예시:
```
/home/youjung/pr/child/mynote.py
```
👉 **정확한 전체 경로를 바로 보여줍니다.**
